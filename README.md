# The Anatomy of Ideas

An interactive atlas of philosophy, psychology, and consciousness. **167 ideas** across
**eleven great questions**, cross-linked to a **425-star constellation** of the thinkers and
schools that shaped them. It is *idea-centric* rather than name-centric: the unit is the
concept, and people orbit the concepts they originated.

The whole thing is a single self-contained HTML file with no backend — `index.html`.
Drop it on any static host and it runs.

---

## Quick start

**Just deploy it:** `index.html` (at the repo root) is prebuilt and standalone — D3 and the
two fonts are inlined, so it makes **zero network calls** at runtime. Upload it anywhere
(Netlify, GitHub Pages, Cloudflare Pages, S3, Vercel). On GitHub Pages, serve from the repo
root and `index.html` is picked up automatically.

**Rebuild it from source:**

```bash
python3 build.py          # -> index.html (repo root)   (needs Python 3, no pip deps)
```

**Test it:**

```bash
npm install               # jsdom + d3, for the headless smoke test
npm test                  # exercises all 5 modes, filters, threads, deep-links, reading store
```

## How it's built

Two data layers, one viewer.

1. **The idea layer** is authored in Python. `concepts.py` holds the base; `expand.py`,
   `expand2.py`, `expand3.py` layer on later rounds (each imports the previous so order is
   fixed). Running `expand3.py` applies all three rounds and writes `ideas.json`. Then
   `merge.py` attaches each concept's ranked thinkers and readings from the `readings*.py`
   files and **fails loudly if any concept lacks readings** or references an unknown id.

2. **The people layer** is `atlas_graph.json` — a star-chart of thinkers and schools grouped
   into West / Psyche / East. It started at 376 nodes and was grown to 425 by the scripts in
   `migrations/`. Those are already applied; the JSON is now the source of truth. (The scripts
   are idempotent if you ever re-run them, but you don't need to.)

3. **The viewer** (`template.html`) gets both JSON blobs spliced in at build time. It's
   vanilla JS + D3 v7, no framework. Cross-linking between layers is by normalized name match
   (with a small alias table and an unambiguous-surname fallback).

### The five modes

- **Questions** — the eleven great questions, with rival answers side by side.
- **Timeline** — every idea at its era, in West/Psyche/East lanes; dot size = how many later
  ideas build on it; hover traces its kin as arcs.
- **Idea Web** — the force-directed graph of concepts, coloured by question.
- **Threads** — the 15 dialectics walked one move at a time (thesis → antithesis → synthesis).
- **Constellation** — the original star-chart of thinkers and schools.

Cross-cutting: a filter rail (question / region / era / reading status), a per-idea
**reading tracker** (unread / reading / read, persisted in `localStorage`), an
**"argues against"** foil on every dossier, and **deep-linkable URLs**
(`#/timeline/idea/The-Burnout-Society`, `#/threads/2.1`).

## Editing the content

- **Add or change an idea:** edit the relevant `expandN.py` (or `concepts.py`) — append a
  tuple `(id, question, tradition, era, gloss, [thinkers])` to its `NEW_C*` list, add any
  links to `NEW_R*`, then add a readings entry in the matching `readings_extra*.py`. Rebuild.
  `merge.py` will refuse to build if the idea has no readings.
- **Add a thinker to the constellation:** add a node + links to `src/atlas_graph.json`
  (see the migration scripts for the shape). Names must match the concept-layer spelling for
  the cross-links to light up.
- Always `npm test` after editing.

## Deploying

It's a single static file at the repo root, so:

```bash
# GitHub Pages: commit index.html, then Settings → Pages → Source = "Deploy from a branch",
#   branch = main, folder = / (root). The prebuilt index.html is served as-is (no CI build).
# Netlify / Cloudflare Pages / Vercel: publish dir = repo root, no build command needed.
```

No environment variables, no server, no database.

Originally built as a private research scaffold, then taken through a launch-readiness pass.

**Done:**

- [x] **Facts verified.** All 167 concepts, 276 thinker attributions, and 292 readings were
      externally checked (dates, titles, authorship). One contested dating was softened; the
      rest held up. Errors may still remain — corrections welcome via GitHub Issues.
- [x] **"How to read this" panel.** States the viewpoint (leans toward detachment), that the
      links are interpretive, and that several debates (consciousness theories, idealism vs
      materialism, free will) are *live*. Contested positions carry a "Contested" tag.
- [x] **Self-hosted assets.** D3 and both fonts are inlined into `index.html` — no Google
      Fonts (GDPR), no CDN single-point-of-failure. Zero runtime network calls.
- [x] **Deploy hygiene.** Title/description/OG tags + favicon in `<head>`; an empty-state
      message when no ideas match the filters; CC BY 4.0 LICENSE + NOTICE for bundled deps.
- [x] **Accessibility baseline.** Keyboard/ARIA/reduced-motion on the chrome; SVG nodes are
      focusable with labels; the "How to read this" dialog is a real focus trap.

**Still open (v2):**

- [ ] **Mobile.** The desktop layout is solid; a phone-first layout (outline instead of the
      force-graph hairball) is not built yet. The page notes it's best on desktop.
- [ ] **Screen-reader navigability of the graphs** (nodes announce, but relationships/arcs
      don't) and a **colour-blind audit** of the 11-hue question encoding.
- [ ] **Representation.** Women are ~7% of thinkers; Japanese, African, Indigenous, and
      Latin-American thought are thin or absent; the map leans toward the detachment thesis it
      was built around. The framing panel owns this; broadening the canon is editorial v2 work.

## Tech notes

- Vanilla JS + D3 v7 and Cormorant Garamond + Inter, all **self-hosted / inlined** into the
  single `index.html`. No build step for the viewer itself beyond JSON + asset injection.
- `localStorage` key `aoi.reading.v1` holds the reading tracker. Per-browser, no sync.
- No analytics, no cookies set by the app, **no runtime network calls at all**.
- Data integrity is enforced at build time: no orphan concepts, full reading coverage, no
  isolated stars.
